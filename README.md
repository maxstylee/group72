# 1. React აპლიკაციის შექმნა

## პროექტის ინიციალიზაცია

გახსენი ტერმინალი და Vite-ის მეშვეობით შექმენი ახალი React პროექტი. დააინსტალირე საჭირო პაკეტები და დარწმუნდი, რომ პროექტი ბრაუზერში შეუფერხებლად ეშვება.

## საქაღალდეების სტრუქტურა

პროექტში შექმენი გამართული და ორგანიზებული არქიტექტურა — მაგალითად, ცალკეული საქაღალდეები კომპონენტებისთვის (components), გვერდებისთვის (pages) და სტატიკური ფაილებისთვის (assets). კოდი უნდა იყოს სუფთა და ადვილად წასაკითხი.

## ბაზისური კომპონენტები

JSX-ის გამოყენებით შექმენი მინიმუმ ორი React კომპონენტი: Header და Footer. ორივე მათგანი შემოიტანე მთავარ გვერდზე და დარწმუნდი, რომ ეკრანზე სწორად აისახება.

# 2. React Hooks (ჰუკები)

## useState

შექმენი კომპონენტი, სადაც useState ჰუკის დახმარებით შეინახავ გარკვეულ სტეიტს (State) — მაგალითად, მომხმარებლის სახელს ან ქაუნთერს (Counter). დაამატე ღილაკი, რომელიც ამ მნიშვნელობას შეცვლის.

## useEffect

იმავე ან სხვა კომპონენტში გამოიყენე useEffect ჰუკი — მაგალითად, გვერდის ჩატვირთვისას კონსოლში გამოიტანე შეტყობინება, ან წამოიღე მონაცემები API-დან.

## React Context

შექმენი Context და მისი მეშვეობით მონაცემები (მაგ. მომხმარებლის ინფორმაცია) რამდენიმე კომპონენტს გაუზიარე props-ების გადაცემის გარეშე.

## Custom Hook

დაწერე საკუთარი (Custom) ჰუკი — მაგალითად, useWindowWidth, რომელიც ეკრანის მიმდინარე სიგანეს აბრუნებს, ან useLocalStorage, რომელიც მონაცემებს ბრაუზერის ლოკალურ მეხსიერებაში ინახავს.

## ოპტიმიზაცია

გამოიყენე useMemo ან useCallback იქ, სადაც დიდი გამოთვლები ან ისეთი ფუნქციები გაქვს, რომლებიც ხშირად ხელახლა იქმნება. კოდში კომენტარის სახით ახსენი, რატომ გახდა ამ ჰუკების გამოყენება საჭირო.

# 3. კომპონენტები და Props

## მონაცემთა გადაცემა

შექმენი მშობელი (Parent) კომპონენტი, რომელშიც იქნება მონაცები (მაგ. პროდუქტების ან სტუდენტების მასივს). ეს სია props-ების სახით გადაეცი შვილ (Child) კომპონენტს.

## მონაცემთა ასახვა

შვილ კომპონენტში მიღებული მონაცემები ეკრანზე გამოიტანე. თითოეული ელემენტის რენდერინგისთვის გამოიყენე .map() მეთოდი და არ დაგავიწყდეს უნიკალური key ატრიბუტის მინიჭება.

## პირობითი რენდერინგი და PropTypes

დაამატე პირობითი რენდერინგი (Conditional Rendering) — მაგალითად, თუ სია ცარიელია, ეკრანზე გამოჩნდეს ტექსტი „მონაცემები არ მოიძებნა“, სხვა შემთხვევაში კი — თავად სია.

# 4. React Router

## როუტინგის გამართვა

დააინსტალირე react-router-dom და პროექტი მინიმუმ სამ გვერდად დაყავი: მთავარი, პროდუქტები და კონტაქტი. თითოეულ გვერდს მიანიჭე საკუთარი უნიკალური URL მისამართი.

## ნავიგაცია

Link კომპონენტის გამოყენებით შექმენი სანავიგაციო მენიუ, რათა გვერდებს შორის გადასვლა ბრაუზერის გადატვირთვის (Reload) გარეშე იყოს შესაძლებელი.

## დინამიური როუტები

დაამატე დინამიური გვერდი (მაგ. /product/:id), სადაც useParams ჰუკის მეშვეობით წაიკითხავ id-ს და ეკრანზე შესაბამისი პროდუქტის დეტალებს გამოაჩენ.

# 5. Redux Toolkit


## სტეიტის მომზადება

დააინსტალირე redux-toolkit. შექმენი გლობალური საცავი (store) და მინიმუმ ერთი სლაისი (slice) — მაგალითად, cartSlice (კალათა) ან userSlice (მომხმარებელი).

## Store-ის დაკავშირება

main.jsx ფაილში, Provider კომპონენტის დახმარებით, store მთელ პროექტთან დააკავშირე.

## სტეიტთან მუშაობა

ნებისმიერ კომპონენტში გამოიყენე useSelector გლობალური სტეიტის წასაკითხად, ხოლო useDispatch — მის შესაცვლელად (მაგალითად: კალათაში პროდუქტის დამატება/წაშლა ან ავტორიზაცია/გამოსვლა).

## ექშენები (Actions)

სლაისში აღწერე მინიმუმ სამი ექშენი — მაგალითად, addItem, removeItem და clearCart — და თითოეული მათგანი გამოიყენე შესაბამის კომპონენტებში.

# 6. ფორმები და ვალიდაცია

## ბიბლიოთეკების ინტეგრაცია

დააინსტალირე react-hook-form და Yup. შექმენი სარეგისტრაციო ფორმა, რომელიც შეიცავს სახელის, ელფოსტისა და პაროლის ველს.

## ვალიდაციის სქემა

Yup-ის მეშვეობით შექმენი ვალიდაციის სქემა, სადაც სახელი იქნება სავალდებულო, ელფოსტა — სწორი ფორმატის, ხოლო პაროლი — მინიმუმ 6-სიმბოლოიანი.

## ფორმის მართვა

ფორმის სამართავად გამოიყენე useForm ჰუკი. თითოეული ველი register ფუნქციით დააკავშირე, ხოლო შეცდომების გამოსატანად formState: { errors } გამოიყენე.
